# 성장 기록·다가오는 일정 다듬기 설계

## 목표

기존 일정과 성장 화면의 정보 구조, 저장 형식, 팝업을 유지하면서 세 가지 불편을 해결한다.

1. iOS에서 일정 탭의 포커스 테두리가 남아 일정과 성장이 동시에 선택된 것처럼 보이는 문제를 없앤다.
2. 최근 성장 카드에서 모든 과거 측정 기록을 날짜별로 찾고 수정할 수 있게 한다.
3. 일정 화면 맨 아래에 오늘부터 가까운 일정 최대 20개를 표시한다.

이 작업은 기존 기능을 교체하는 재설계가 아니다. 현재 캘린더, 성장 그래프, 성장 기록 목록, 추가·수정 팝업, Supabase 저장 흐름을 그대로 두고 발견성과 조작성을 보완한다.

## 현재 상태와 근거

- 실제 로그인된 별빛 밤 화면에서 성장 탭만 `active=true`이고 일정 탭은 `active=false`였다.
- 첨부된 iPhone 화면에서 일정 탭 위에 남은 파란 테두리는 터치 후 유지된 전역 `focus-visible` 링이다.
- 최근 성장 그래프의 각 점에는 기존 기록 ID와 키보드 접근성이 연결되어 있고, 점을 누르면 해당 기록이 기존 `openGrowthDialog(entry)`로 열린다.
- 과거 기록 수정 자체는 가능하지만 그래프 점의 터치 영역이 작고 날짜별 목록이 없어 기능을 발견하기 어렵다.
- 성장 기록 추가·수정 팝업은 이미 동일한 `saveGrowthEntry()`와 `growth_entries` upsert 흐름을 사용한다.
- `다가오는 일정`은 별도 설계 문서 `2026-07-22-upcoming-events-list-design.md`가 있으나 아직 화면과 렌더링 코드가 구현되지 않았다.

## 보존 원칙

- 기존 사용자 데이터와 Supabase 테이블·RLS·저장 형식을 변경하지 않는다.
- 기존 일정 추가·수정·삭제 팝업과 `openEventDialog(event)`를 재사용한다.
- 기존 성장 추가·수정·삭제 팝업과 `openGrowthDialog(entry)`, `saveGrowthEntry()`를 재사용한다.
- 최근 성장 그래프와 그래프 점 편집 기능을 유지한다.
- 기존 성장일기 목록과 카테고리 필터를 유지한다.
- 기존 SUIT Variable 단일 텍스트 폰트와 별빛 밤·포근한 숲 테마를 유지한다.
- 인증, 사진 저장, 가족·아기 선택, AI, 알림 로직을 변경하지 않는다.
- 사용자 데이터를 검증 목적으로 생성·수정·삭제하지 않는다.
- `HANDOFF.md`와 `.superpowers/`는 수정하거나 커밋하지 않는다.

## 1. 일정·성장 탭 선택 상태

### 문제

모바일 Safari/PWA에서 터치한 탭이 포커스를 유지하면 전역 `focus-visible` 링이 남는다. 다른 탭으로 이동한 뒤에도 이전 탭에 파란 외곽선이 보여 실제 활성 배경과 혼동된다.

### 설계

- `.view-tab`의 기존 `active` 클래스가 유일한 선택 상태가 되도록 한다.
- 터치 또는 펜 입력으로 탭을 선택한 경우 클릭 처리가 끝난 뒤 해당 버튼의 DOM 포커스를 해제한다.
- 마우스 클릭과 키보드 Tab/Enter/Space 흐름은 기존 포커스 표시를 유지한다.
- 탭 높이, 모서리, 배경색은 현재 디자인 토큰을 유지한다.
- CSS에서 포커스를 전면 제거하지 않는다. 입력 방식에 따른 포커스 정리만 추가한다.

### 구현 위치

탭 전환을 소유한 `app.js`의 기존 `.view-tab` 이벤트 연결부에 작은 입력 방식 처리만 추가한다. 별도 탭 전용 스크립트나 임시 CSS 파일은 만들지 않는다.

## 2. 최근 성장 카드와 과거 측정 기록

### 정보 구조

현재 최근 성장 카드의 제목, 최신 날짜, `기록 추가`, `기록 수정`, 그래프를 유지한다. 그래프 아래 안내 문구 다음에 `측정 기록` 영역을 추가한다.

기록 영역은 다음 순서로 구성한다.

1. 제목 `측정 기록`
2. 보조 문구 `최근 기록부터 확인하고 수정할 수 있어요`
3. 현재 표시 개수
4. 날짜별 기록 행
5. 기록이 5개를 넘으면 `전체 기록 보기` 또는 `최근 기록만 보기` 토글
6. `새 측정 기록` 버튼

### 기록 행

- 최신순으로 정렬한다.
- 날짜와 선택적 시간을 표시한다.
- 키, 몸무게, 머리둘레를 같은 순서로 표시한다.
- 값이 없는 항목은 `—`로 표시한다.
- 행 전체가 버튼이며 오른쪽에 `수정`을 명확히 표시한다.
- 접근 가능한 이름에는 날짜와 존재하는 측정값, `기록 수정`을 포함한다.
- 최소 터치 높이는 56px로 한다.
- 기본 상태에서는 최근 5개만 표시한다.

행을 누르면 해당 ID의 기존 성장 기록을 찾아 `openGrowthDialog(entry)`로 연다. 목록에서 별도의 저장·삭제 함수를 만들지 않는다.

### 추가·수정 팝업 다듬기

- 팝업 구조와 모든 기존 입력 필드를 유지한다.
- 성장 카테고리에서는 키·몸무게·머리둘레 영역의 시각적 우선순위를 높인다.
- `기록 제목`은 `기록 이름 (선택)`으로 명확히 표시한다. 비워 두면 기존 기본 제목 생성 로직을 그대로 사용한다.
- 날짜와 시간을 측정값 가까이에 유지한다.
- 추가 시 `기록하기`, 수정 시 `수정하기`와 삭제 버튼을 유지한다.
- 유효성·오류 메시지는 팝업을 닫지 않고 현재 저장 흐름을 따른다.

### 상태와 갱신

- 펼침 상태는 화면 세션 동안의 UI 상태로만 관리하며 DB나 로컬 저장소에 기록하지 않는다.
- `growthEntries`가 변경되거나 아기가 바뀌면 그래프와 측정 기록 목록을 같은 데이터 집합에서 다시 그린다.
- 표시 데이터의 기준은 기존 `measurementEntries()`로 통일한다.

### 구현 위치

- 목록 렌더링과 클릭 처리는 기존 인라인 카드를 소유한 `growth-inline-chart.js`에 둔다.
- 스타일은 `growth-inline-chart.css`와 `growth-inline-approved-polish.css`에서 정리하되 중복 규칙은 최소화한다.
- 기존 `growth-measurements.js`의 별도 곡선 팝업과 기록 목록은 제거하지 않는다.

## 3. 다가오는 일정

`docs/superpowers/specs/2026-07-22-upcoming-events-list-design.md`의 필터·정렬·표시 규칙을 그대로 구현한다.

### 배치

일정 화면의 선택 날짜 일정 영역 다음, 즉 `#calendarView` 맨 아래에 `다가오는 일정` 섹션을 배치한다.

### 표시 규칙

- 오늘 진행 중이거나 오늘 이후 시작하는 가족 일정만 포함한다.
- 이미 종료된 일정과 유효하지 않은 날짜는 제외한다.
- 가까운 날짜, 진행 중인 여러 날 일정, 시간순 기준으로 정렬한다.
- 종일 일정은 같은 날의 시간 지정 일정 뒤에 둔다.
- 최대 20개만 표시한다.
- 날짜·요일, 시간 또는 `종일`, 제목, 담당 가족, 여러 날 기간을 표시한다.
- 항목 전체를 누르면 기존 `openEventDialog(event)`를 연다.
- 일정이 없으면 `다가오는 일정이 없어요`를 표시한다.

### 구현 경계

- `upcomingEvents(events, todayKey, limit = 20)`는 필터·정렬·제한만 담당하는 순수 함수로 둔다.
- `renderUpcomingEvents()`는 DOM과 기존 편집 팝업 연결만 담당한다.
- 일정 저장 후 기존 `render()` 경로에서 자동으로 갱신한다.
- 별도 데이터 요청이나 저장소를 추가하지 않는다.

## 반응형과 시각 원칙

- 390×844과 430×932에서는 성장 기록 행과 다가오는 일정 행을 한 열로 유지한다.
- 768×1024와 1440×900에서도 시간 순서를 흐리지 않도록 목록은 한 열로 유지하고 최대 너비만 제한한다.
- 긴 제목은 본문 영역 안에서 말줄임하거나 자연스럽게 두 줄까지 표시해 가로 스크롤을 만들지 않는다.
- `documentElement.scrollWidth`가 viewport를 넘지 않아야 한다.
- 별빛 밤에서는 현재 네이비 카드와 절제된 파란 선택색을 유지한다.
- 포근한 숲에서는 기존 크림·초록 토큰을 사용한다.
- 기존 이모지 체계 외에 장식용 이모지를 추가하지 않는다.

## 접근성

- 탭의 `active` 상태와 키보드 포커스를 서로 다른 시각 신호로 유지한다.
- 키보드 사용자는 모든 탭, 성장 기록 행, 다가오는 일정 행에 접근할 수 있어야 한다.
- 성장 기록과 일정 행의 접근 가능한 이름에 핵심 날짜와 내용을 포함한다.
- 새 섹션은 `aria-labelledby`로 제목과 연결한다.
- 기존 `focus-visible` 규칙을 유지하고 터치 입력에서만 잔류 포커스를 정리한다.

## 오류 처리

- 기록 ID를 찾지 못하면 팝업을 열지 않고 현재 화면을 유지한다.
- 성장 또는 일정 저장 실패 시 기존 토스트와 팝업 상태를 유지한다.
- 일부 측정값이 없어도 해당 기록 행은 표시한다.
- 다가오는 일정 렌더링 실패가 기존 캘린더와 저장 흐름에 영향을 주지 않게 순수 선택 함수와 DOM 렌더러를 분리한다.

## 테스트

### 탭

- 터치·펜 입력 후 탭 포커스를 정리한다.
- 마우스와 키보드 입력에서는 포커스를 강제로 제거하지 않는다.
- 활성 클래스는 한 탭에만 남는다.

### 성장 기록

- 측정 기록을 최신순으로 표시한다.
- 기본 5개 제한과 전체 펼치기/접기를 검증한다.
- 일부 값이 없는 기록을 `—`로 표시한다.
- 기록 행과 그래프 점이 동일한 기존 기록 수정 팝업을 연다.
- 새 측정 기록 버튼이 기존 성장 추가 팝업을 연다.
- 추가·수정·삭제 후 그래프와 목록이 함께 갱신된다.

### 다가오는 일정

- 오늘·미래·진행 중 일정 포함, 종료 일정 제외를 검증한다.
- 날짜·진행 중·시간·종일·안정 정렬을 검증한다.
- 최대 20개 제한과 빈 상태를 검증한다.
- 항목 클릭이 기존 일정 수정 팝업을 연다.

### 전체 검증

- `npm test`
- `npm run check`
- `git diff --check`
- 별빛 밤과 포근한 숲에서 390×844, 430×932, 768×1024, 1440×900 확인
- 실제 로그인 화면에서 일정·성장 탭 반복 전환 후 잔류 포커스가 보이지 않는지 확인
- 검증 중 기존 사용자 데이터를 생성·수정·삭제하지 않는다.
