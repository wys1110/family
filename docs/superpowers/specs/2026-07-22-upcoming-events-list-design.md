# 다가오는 일정 목록 설계

## 목표

일정 화면의 맨 아래에서 오늘 진행 중이거나 앞으로 시작하는 가족 일정을 가까운 순서대로 최대 20개까지 확인한다. 기존 선택 날짜 일정과 캘린더는 유지하고, 새 목록은 읽기 전용 탐색 화면으로만 추가한다.

## 범위

- 대상 데이터는 기존 `state.events`에 들어 있는 가족 일정이다.
- 대한민국 공휴일처럼 화면에서 임시 생성되는 일정은 포함하지 않는다.
- 인증, Supabase, 로컬 저장, 일정 생성·수정·삭제 형식은 변경하지 않는다.
- `HANDOFF.md`와 `.superpowers/`는 수정하거나 커밋하지 않는다.

## 화면 구성

`#calendarView`의 선택 날짜 일정 영역 다음에 `다가오는 일정` 섹션을 배치한다.

- 제목: `다가오는 일정`
- 보조 문구: `오늘부터 가까운 순서`
- 개수 배지: 실제 표시 개수
- 목록: 한 열의 compact 타임라인, 최대 20개
- 빈 상태: `다가오는 일정이 없어요`

각 항목에는 다음 내용을 표시한다.

- 시작 날짜와 요일
- 시간 또는 `종일`
- 일정 제목
- 담당 가족 구성원
- 여러 날 일정이면 기존 `formatEventRange()`의 기간

항목 전체를 버튼으로 제공하고 누르면 기존 `openEventDialog(event)`를 호출한다. 별도의 저장 또는 상세 화면은 만들지 않는다.

## 포함 조건

오늘 날짜 키를 `todayKey`라고 할 때 아래 조건을 만족하는 일정만 포함한다.

```text
(event.endDate || event.date) >= todayKey
```

따라서 다음을 포함한다.

- 오늘 시작하거나 오늘 예정된 일정
- 미래에 시작하는 일정
- 과거에 시작했지만 오늘 또는 미래에 끝나는 여러 날 일정

날짜 키는 `YYYY-MM-DD` 형식이며 `dateKey(parseDate(key)) === key`를 만족해야 유효한 것으로 본다. 이미 종료된 일정과 이 검증을 통과하지 못한 일정은 제외한다.

## 정렬 및 제한

각 일정의 정렬 기준일은 `max(event.date, todayKey)`로 계산한다. 과거에 시작해 진행 중인 일정은 오늘 일정처럼 가장 위에 놓인다.

정렬 우선순위는 다음과 같다.

1. 정렬 기준일 오름차순
2. 같은 날이면 진행 중인 여러 날 일정 우선
3. 같은 조건이면 실제 시작일 오름차순
4. 같은 날의 시간 있는 일정은 `HH:mm` 오름차순
5. 종일 일정은 시간 일정 뒤
6. 제목, ID 순으로 안정 정렬

정렬이 끝난 뒤 앞에서 20개만 표시한다. 선택 날짜 영역과 다가오는 일정 목록에서 오늘 일정이 중복될 수 있으며, 이는 선택 날짜 상세와 전체 미래 탐색이라는 서로 다른 목적을 유지하기 위해 허용한다.

## 구현 경계

`app.js`에 다음의 작은 단위를 둔다.

- `upcomingEvents(events, todayKey, limit = 20)`: 필터·정렬·제한만 담당하는 순수 함수
- `renderUpcomingEvents()`: DOM 렌더링과 기존 편집 팝업 연결 담당

`render()`와 일정 변경 후 다시 그리는 경로에서 `renderUpcomingEvents()`를 호출한다. 필터·정렬 규칙은 렌더링 함수 안에 중복 작성하지 않는다.

HTML에는 섹션과 빈 목록 컨테이너만 추가하고, 스타일은 기존 일정 카드 토큰과 SUIT Variable을 재사용한다. 화면별 임시 스타일 파일은 만들지 않는다.

## 반응형·접근성

- 390×844부터 1440×900까지 한 열의 시간 순서를 유지한다.
- 날짜 열은 고정 폭, 본문은 `minmax(0, 1fr)`로 구성해 긴 제목이 화면을 밀어내지 않게 한다.
- 항목의 최소 터치 높이는 52px로 한다.
- 섹션은 `aria-labelledby`로 제목과 연결한다.
- 각 항목의 접근 가능한 이름에는 날짜, 제목, 시간 또는 종일 여부를 포함한다.
- 기존 `focus-visible` 규칙을 유지하고 문서 가로 스크롤을 만들지 않는다.
- 모든 텍스트는 자체 호스팅된 SUIT Variable을 사용하고 이모지를 새로 추가하지 않는다.

## 오류 및 빈 상태

- 일정이 0개면 빈 상태만 표시한다.
- 일부 일정의 `endDate`가 없으면 `date`를 종료일로 사용한다.
- 유효하지 않은 날짜 데이터는 목록에서 제외하되 기존 데이터는 수정하거나 삭제하지 않는다.
- 목록 렌더링 실패가 일정 저장 로직에 영향을 주지 않도록 순수 선택 함수와 DOM 렌더링을 분리한다.

## 테스트 및 검증

- 오늘 일정, 미래 일정, 종료된 일정 필터링
- 진행 중인 여러 날 일정 포함 및 우선 정렬
- 같은 날 시간순과 종일 일정 후순위
- 동일 조건에서 안정 정렬
- 20개 제한
- HTML 섹션·제목·개수 배지·빈 상태 계약
- 항목 클릭 시 기존 일정 편집 팝업 연결
- `npm test`, `npm run check`, `git diff --check`
- 별빛 밤과 포근한 숲에서 390×844, 430×932, 768×1024, 1440×900 실제 로그인 화면 확인
- 각 해상도에서 `documentElement.scrollWidth <= documentElement.clientWidth` 확인
