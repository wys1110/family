# PC 로그아웃 버튼 설계

## 배경

현재 로그아웃 기능은 상단의 원형 계정 버튼을 눌러 계정 팝업을 연 뒤에만 사용할 수 있다. 모바일에서는 이 흐름이 비교적 잘 보이지만, 넓은 PC 화면에서는 계정 버튼이 헤더 오른쪽 끝에 작은 원형으로 표시되어 로그아웃 진입점을 알아보기 어렵다.

## 목표

- PC 화면에서 로그인 사용자가 로그아웃 기능을 즉시 발견할 수 있게 한다.
- 모바일의 현재 계정 팝업과 로그아웃 흐름은 변경하지 않는다.
- 로그인하지 않은 상태나 Supabase 미연결 상태에서는 로그아웃 버튼을 노출하지 않는다.

## UI 설계

- 헤더의 기존 계정 버튼 옆에 `로그아웃` 텍스트 버튼을 추가한다.
- 기본 상태에서는 숨기고, 화면 폭이 768px 이상이면서 인증 세션이 있을 때만 표시한다.
- 기존 원형 계정 버튼은 가족 초대와 계정 상태 확인을 위해 유지한다.
- 버튼 스타일은 현재 헤더 및 계정 버튼 색상 체계와 어울리는 보조 버튼으로 한다.

## 동작

- 버튼을 누르면 기존과 동일하게 `state.supabase.auth.signOut()`을 호출한다.
- 중복 구현을 피하기 위해 계정 팝업과 PC 헤더가 공통 로그아웃 함수를 사용한다.
- 로그아웃 처리 중에는 버튼을 비활성화하고, 실패하면 로그인 상태를 유지한 채 오류 안내를 표시한다.
- 인증 상태가 바뀔 때 버튼의 노출 상태를 다시 계산한다.

## 검증

- 정적 계약 테스트로 헤더 버튼, 공통 로그아웃 함수, 데스크톱 전용 CSS를 확인한다.
- 767px 이하에서는 숨김, 768px 이상 로그인 상태에서는 표시되는지 브라우저에서 확인한다.
- PC 헤더 버튼과 기존 계정 팝업 버튼이 모두 같은 로그아웃 동작을 수행하는지 확인한다.
- 기존 전체 테스트와 정적 검사를 다시 실행한다.

## 제외 범위

- 로그인 방식 변경
- 계정 팝업 재설계
- 모바일 헤더 구조 변경
- Supabase 인증 정책 변경
