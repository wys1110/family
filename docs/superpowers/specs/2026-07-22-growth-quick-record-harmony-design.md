# 직수·기저귀 빠른 기록 UI 조화 설계

## 목표

`직수를 바로 기록해요` 빠른 기록 창을 옆의 기저귀 빠른 기록 창과 같은 디자인 체계로 정리한다. 제목, 설명, 카드, 간격과 색상 밀도를 통일하고 직수 선택 카드를 누르면 현재 시간으로 즉시 저장되게 한다.

## 현재 확인된 불일치

로그인된 455×812 화면에서 실제 computed style을 비교했다.

| 요소 | 직수 | 기저귀 |
| --- | --- | --- |
| 다이얼로그 폭 | 560px | 500px |
| 제목 | 34px / 790 | 18px / 700 |
| 안내문 | 12px | 11px |
| 선택 구조 | 전용 3열 카드 + 별도 저장 버튼 | 2열 82px 즉시 기록 카드 |
| 상세 입력 버튼 | 12px / 56px | 15px / 54px |

직수만 별도 시각 체계를 사용해 같은 `기록하기` 섹션의 기저귀 동작과 이질감이 발생한다.

## 승인된 동작

- `왼쪽 10분`, `오른쪽 10분`, `양쪽 20분` 카드는 누르는 즉시 현재 시간으로 기존 빠른 기록 저장 함수를 호출한다.
- 선택 후 다시 누르는 별도 `기록하기` 버튼은 제거한다.
- 직수·유축모유·분유 전환은 유지한다.
- 유축모유와 분유의 양 조절·확인 흐름은 변경하지 않는다.
- `직접 상세 입력`은 유지하고 기존 성장 기록 다이얼로그를 연다.
- 저장 데이터 모양, Supabase 호출, 완료 안내와 기록 갱신은 변경하지 않는다.

## 시각 시스템

- 직수 상태에서도 기저귀와 동일한 기본 `quick-log-dialog` 폭과 패딩을 사용한다.
- 제목은 18px/700, 설명은 11px와 동일 행간을 사용한다.
- 직수 프리셋은 기저귀가 사용하는 2열 그리드와 최소 높이 82px 카드를 재사용한다.
- 카드 배경, 테두리, 라운드, 그림자와 본문 색상은 기저귀 프리셋의 의미 토큰을 그대로 사용한다.
- 직수 카드 안의 주 문구와 보조 문구 크기·굵기도 기저귀 카드와 동일하게 맞춘다.
- 직수·유축모유·분유 전환 탭은 기능 구분에 필요한 만큼만 유지하며, 높이와 글자 크기는 다른 빠른 기록 컨트롤보다 커지지 않게 한다.
- 포근한 숲과 별빛 밤의 기존 테마 토큰을 사용하고 새로운 고정 색상은 추가하지 않는다.
- SUIT Variable 단일 텍스트 폰트 정책을 유지한다.

## 구현 방향

- `adaptive-feeding.js`의 직수 전용 선택·확인 UI를 표준 빠른 프리셋 마크업과 즉시 저장 핸들러로 교체한다.
- 직수 전용으로 다이얼로그와 제목을 과도하게 키우는 CSS를 제거하거나 표준 빠른 기록 규칙으로 재정의한다.
- 기존 기저귀 UI를 복제하지 않고 공통 클래스와 토큰을 재사용한다.
- 변경 모듈의 캐시 버전을 갱신한다.

## 접근성과 오류 처리

- 각 카드의 접근 가능한 이름에 방향과 시간을 포함한다.
- 카드의 최소 터치 영역 48px 이상과 focus-visible을 유지한다.
- 저장 중 중복 탭 방지는 기존 빠른 기록 저장 상태를 그대로 사용한다.
- 저장 실패 시 기존 오류 토스트를 사용하고 다이얼로그 상태를 임의로 초기화하지 않는다.

## 검증

- 동작 테스트에서 각 직수 카드를 누르면 정확한 프리셋 하나가 즉시 저장되는지 확인한다.
- 별도 직수 확인 버튼이 렌더링되지 않는지 확인한다.
- 직수·기저귀의 다이얼로그 제목, 안내문, 카드 그리드와 상세 입력 computed style을 비교한다.
- 별빛 밤과 포근한 숲에서 실제 로그인 화면을 열고 저장 버튼은 누르지 않는 구조 검증과 테스트 기반 저장 동작 검증을 병행한다.
- `npm test`, `npm run check`, `git diff --check`를 실행한다.
