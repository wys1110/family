# AI 육아 도우미 검색 근거·쉬운 설명 설계

## 목표

AI 육아 도우미의 일반 질문과 수유·수면 전략을 실시간 인터넷 자료에 근거해 만들고, 보호자가 근거를 직접 확인할 수 있게 한다. 답변은 초등학생도 한 번에 이해할 수 있는 한국어로 작성한다.

『삐뽀삐뽀 119 소아과』는 저자 하정훈 소아청소년과 전문의의 공개 자료와 공식 유튜브 채널을 신뢰 가능한 보조 출처로 취급한다. 앱에 책 원문이 없으므로 책의 특정 문장이나 페이지를 인용했다고 표현하지 않는다.

## 범위

- 일반 육아 질문 답변
- 수유·수면 전략 생성 및 자동 갱신
- 실제 웹·유튜브 출처 표시
- 쉬운 설명 형식과 의료 안전 경계
- AI Function 실패 원인을 사용자가 이해할 수 있는 범주로 표시

책 전체 저장, 영상 자막의 장기 보관, 의학적 진단·처방, 약 이름이나 용량 추천은 범위에서 제외한다.

## 출처 정책

출처는 다음 순서로 우선한다.

1. 질병관리청, 보건복지부, 소방청 등 국내 공공기관
2. 대한소아청소년과학회 등 전문 학회
3. WHO, CDC, AAP와 AAP의 HealthyChildren, NHS 등 공신력 있는 해외 기관
4. 대학병원·어린이병원 공식 자료
5. 소아청소년과 전문의가 직접 운영하거나 출연한 검증된 유튜브 채널

유튜브에서는 하정훈의 `삐뽀삐뽀 119 소아과` 공개 채널, 공공기관·병원 공식 채널, 소아청소년과 전문의의 신원이 확인되는 채널만 허용한다. 일반 육아 브이로그, 판매 링크 중심 영상, 협찬·광고성 콘텐츠, 출연자의 전문성을 확인할 수 없는 영상은 근거로 사용하지 않는다.

초기 웹 허용 목록은 `kdca.go.kr`, `mohw.go.kr`, `pediatrics.or.kr`, `nfa.go.kr`, `who.int`, `cdc.gov`, `aap.org`, `healthychildren.org`, `nhs.uk`로 고정한다. 병원이나 학회 도메인은 운영자가 공식 소유와 의료 전문성을 확인한 뒤 목록에 추가한다. 초기 유튜브 허용 목록에는 확인된 하정훈 채널 ID `UC6t0ees15Lp0gyrLrAyLeJQ`를 넣고, 다른 공식기관·전문의 채널도 채널 ID를 확인한 뒤 명시적으로 추가한다. 채널을 확인할 수 없는 개별 영상은 사용하지 않는다.

공식 지침과 전문의 영상이 충돌하면 공식 지침을 우선한다. 출처가 오래되었거나 게시일을 확인할 수 없으면 최신 공식 자료로 교차 확인한다.

## 데이터 흐름과 개인정보 경계

검색은 Gemini 2.5 Flash의 Google Search grounding을 사용한다.

1. 서버가 질문의 주제, 아기의 월령 범위, 수유·수면 같은 일반 범주만으로 검색용 입력을 만든다.
2. 검색용 입력에서 아기·부모 이름, 계정 식별자, 정확한 생활 일정, 자유 메모, 전화번호, 이메일, URL을 제외하거나 마스킹한다.
3. Gemini가 웹 검색을 수행하고 답변 초안과 `groundingMetadata`를 반환한다.
4. 서버가 실제 검색 메타데이터에서 제목과 URL을 추출하고 허용 가능한 출처만 남긴다.
5. 서버가 검색 근거와 기존의 비식별 가족 문맥을 결합해 최종 답변 또는 전략을 만든다.
6. 클라이언트는 답변과 함께 최대 5개의 출처를 링크로 표시한다.

검색어와 검색 메타데이터는 디버그 로그나 데이터베이스에 저장하지 않는다. 전략에 저장하는 출처는 공개 URL, 제목, 매체 유형, 확인 시각뿐이다.

Google 검색 메타데이터가 중간 리디렉션 URL을 반환하면 서버는 HTTPS URL만 한 번 해석한다. 최종 호스트가 사설 IP, 로컬 주소 또는 허용 목록 밖이면 폐기한다. 유튜브 영상은 최종 영상 URL과 채널 ID가 모두 확인될 때만 허용한다.

## 서버 구조

### Gemini 전송 계층

기존 문자열 반환 대신 다음 결과를 반환한다.

```ts
type GroundedResult = {
  text: string;
  sources: Array<{
    title: string;
    url: string;
    type: "web" | "youtube";
  }>;
  grounded: boolean;
};
```

`grounding: true`인 호출에는 `tools: [{ google_search: {} }]`를 추가한다. `groundingChunks`에서 실제 출처만 추출하며 모델이 본문에 임의로 쓴 URL은 출처로 인정하지 않는다.

### 도메인 계층

- 검색용 문맥을 만드는 순수 함수
- 전화번호·이메일·URL과 명시적 식별자를 제거하는 함수
- 허용 출처와 유튜브 출처를 분류하는 함수
- 쉬운 설명 규칙을 포함하는 공통 시스템 지침
- 긴급 신호 감지는 기존처럼 Gemini 호출보다 먼저 실행

### 응답 계약

일반 질문 응답은 `answer`, `urgent`, `sources`, `grounded`를 반환한다.

전략의 `content`에는 기존 필드와 함께 `sources`를 추가한다. 출처는 모델이 생성한 JSON이 아니라 검색 메타데이터에서 서버가 주입한다.

검색이 필요한 일반 답변에서 허용된 출처가 하나도 없으면 한 번만 더 공식 출처 중심으로 검색한다. 두 번째에도 근거가 없으면 확인되지 않은 의학 정보를 생성하지 않고, 출처를 확인하지 못했다는 안내와 소아청소년과 상담 권고를 반환한다.

## 쉬운 설명 형식

일반 답변은 다음 순서를 기본으로 한다.

1. `한 줄 결론`
2. `지금 할 일`
3. `지켜볼 것`
4. `병원에 갈 때`
5. `참고한 자료`

한 문장은 가능하면 25자 안팎으로 짧게 쓴다. 어려운 의학 용어는 처음 나올 때 괄호로 쉬운 뜻을 붙인다. 숫자나 시간 기준이 출처·아기 상태에 따라 달라질 때는 단정하지 않고 조건을 함께 설명한다.

전략 JSON의 각 항목도 한 항목에 한 행동만 담는다. 관찰된 기록과 일반 의학 원칙을 섞지 않고 별도 항목으로 나눈다.

## 의료 안전

- 진단이나 처방을 하지 않는다.
- 약 이름이나 용량을 정하지 않는다.
- 숨쉬기 어려움, 청색증, 의식 저하, 경련, 심한 탈수 등 기존 긴급 신호는 검색을 기다리지 않고 즉시 119 또는 응급실을 안내한다.
- 보호자가 위급하다고 느끼면 데이터가 부족해도 즉시 의료기관에 연락하도록 안내한다.
- 수면 전략은 안전한 수면 환경을 해치거나 보호자에게 위험한 행동을 권하지 않는다.

## 화면 표시

채팅 답변과 전략 카드 아래에 `참고한 자료` 영역을 둔다. 각 출처는 제목, 기관 또는 채널명, 웹·유튜브 구분을 표시하고 새 창으로 연다. 링크에는 `noopener noreferrer`를 적용한다.

검색 중에는 `공식 자료를 확인하고 있어요`라고 표시한다. 출처가 없는 기존 확정 전략은 그대로 보여주되 `이 전략은 출처 표시 기능 적용 전에 생성됐어요`라고 구분한다.

## 오류 처리

현재의 단일 `AI Function 설정과 연결을 확인해 주세요` 메시지를 다음 범주로 나눈다.

- 로그인 만료: 다시 로그인 안내
- 네트워크 실패: 연결 확인 후 재시도 안내
- Gemini 할당량·일시 오류: 잠시 뒤 재시도 안내
- 검색 근거 없음: 질문을 더 구체적으로 쓰거나 의료진 상담 안내
- 서버 설정 누락: 관리자 확인이 필요한 오류로 표시

클라이언트에는 비밀값, 내부 응답 전문, 데이터베이스 오류 내용을 노출하지 않는다. 서버 로그에는 정해진 오류 코드만 남긴다.

## 테스트

- 검색 요청에 `google_search` 도구가 포함되는지 검증
- 검색 메타데이터의 웹·유튜브 출처 추출 검증
- 모델이 본문에 임의 URL을 써도 출처로 채택하지 않는지 검증
- 개인정보가 검색용 입력에서 제거되는지 검증
- 쉬운 설명 지침과 답변 순서 검증
- 긴급 질문이 검색과 Gemini를 우회하는지 회귀 검증
- 전략 출처가 서버에서 주입되고 DB에 보존되는지 검증
- 출처 링크 HTML 이스케이프와 `noopener noreferrer` 검증
- 로그인·네트워크·할당량 오류 메시지 분기 검증
- 모바일과 PC에서 출처 영역이 읽기 쉽게 표시되는지 브라우저 검증

## 배포와 검증

Edge Function을 먼저 배포하고 기존 응답 계약과의 하위 호환성을 확인한 뒤 Pages를 배포한다. 배포 후 로그인한 가족 계정으로 일반 질문, 수유 전략, 수면 전략을 각각 한 번 실행해 출처 링크와 쉬운 설명 형식을 확인한다. 긴급 문구는 Gemini를 호출하지 않는지 서버 결과로 확인한다.

Google Search grounding은 검색 사용량에 따른 비용과 응답 지연이 생길 수 있으므로, 실제 검색을 수행한 요청 수와 실패 코드만 집계한다. 질문 내용과 가족 문맥은 분석 로그에 남기지 않는다.
