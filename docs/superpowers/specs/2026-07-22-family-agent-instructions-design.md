# Family 에이전트 지침 설계

## 목표

Codex와 Claude Code가 같은 짧은 기준을 따르면서 `family` 저장소를 안전하게 변경한다. 사용자가 ChatGPT로 작은 단위의 구현을 자주 가져오는 흐름을 지원하고, 로컬 에이전트는 실제 코드와 Git 상태를 확인한 뒤 필요한 검증을 담당한다.

## 파일 구성

- `AGENTS.md`: 프로젝트별 단일 기준 문서
- `CLAUDE.md`: 내용을 복제하지 않고 `AGENTS.md`를 불러오는 Claude Code 진입점

## 지침 범위

`AGENTS.md`는 다음 정보만 담는다.

1. 프로젝트 구조와 실행·검증 명령
2. 최소 변경 및 토큰 절약 원칙
3. ChatGPT 결과물을 이어받는 방법
4. 가족 데이터, Supabase RLS, 비밀값과 AI 안전 경계
5. Vanilla JS/CSS, 자산 로딩 순서, 스키마·마이그레이션 변경 규칙
6. 기능 브랜치, 검증, PR, 사용자 승인 후 merge 흐름
7. 간결한 완료 보고 형식

README에 이미 있는 기능 목록과 상세 배포 절차는 반복하지 않고 링크와 핵심 금지사항만 유지한다.

## 토큰 절약형 작업 방식

- 명확한 소규모 요청은 긴 계획 없이 관련 파일만 찾아 바로 처리한다.
- `rg`와 Git diff로 범위를 좁히고 전체 저장소나 전체 파일을 불필요하게 출력하지 않는다.
- 한 번에 하나의 작은 수직 변경만 수행하며 추측성 기능과 선제적 리팩터링을 금지한다.
- ChatGPT가 만든 코드나 설명은 현재 checkout의 사실로 간주하지 않는다. 관련 파일, 연결 지점, `git diff`와 테스트로 확인한다.
- 외부 AI에 저장소 파일, 개인정보 또는 비밀값을 자동 전송하지 않는다.
- 결과는 `변경 / 검증 / 남은 것` 중심으로 짧게 보고한다.
- RLS, 인증, migration, Edge Function, 비밀값, 의료 안전과 배포는 토큰 절약보다 정확성과 검증을 우선한다.

## 안전 경계

- 가족 간 데이터 격리, 개인 공간, 비공개 사진 정책을 약화하지 않는다.
- `service_role`, Gemini, VAPID, Cron 및 OAuth 비밀값을 브라우저 코드, Git, 로그, 이슈 또는 채팅에 넣지 않는다.
- Gemini 호출은 Edge Function 경계를 유지하고, 육아 답변에서 진단·처방·약물 용량과 불필요한 개인 식별자를 제외한다.
- Supabase migration, secret, cron 또는 Edge Function 배포는 사용자의 명시적 요청 없이 실행하지 않는다.
- 인증 세션 없이 로그인 이후 화면을 실제 검증했다고 주장하지 않는다.

## Git 및 검증

- `main`에서 직접 작업하거나 push하지 않는다.
- 기능 브랜치에서 변경하고 `npm test`, `npm run check`, `git diff --check`를 실행한 뒤 PR을 만든다.
- merge는 사용자 승인 후 진행한다.
- 코드 merge, GitHub Pages 배포, Supabase 반영 상태를 서로 구분해 보고한다.

## 성공 기준

- 두 도구가 하나의 기준 문서를 사용한다.
- 지침이 상위 `/Users/yongseokwon/dev/AGENTS.md`를 반복하지 않고 `family` 고유 위험만 보완한다.
- 문서가 짧고 실행 가능하며 현재 저장소 명령·구조와 일치한다.
- Markdown 링크와 테스트·정적 검사가 통과한다.
