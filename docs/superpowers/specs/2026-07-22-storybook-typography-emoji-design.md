# Family 동화책 타이포그래피·이모지 설계

## 목표

`family` 앱을 감성적인 동화책처럼 따뜻하게 보이게 하되, 가족 일정과 육아 기록을 빠르게 읽고 조작하는 앱의 기능성은 유지한다. 별빛 밤 테마를 우선 기준으로 삼고 다른 테마에서 가독성과 레이아웃이 깨지지 않는지 회귀 확인한다.

## 디자인 원칙

- 장식보다 의미 전달을 우선한다. 이모지는 탐색 표지판과 상태 신호로 사용한다.
- 한 화면에는 큰 손그림 아이콘 1~2개만 두고, 섹션과 버튼에는 의미가 분명한 기본 이모지를 하나씩 사용한다.
- 카드 하나에는 장식 이모지를 최대 2개만 사용한다.
- AI 답변, 의료 안전 문구, 날짜, 수유량, 차트 축과 긴 설명 문장 안에는 장식 이모지를 넣지 않는다.
- 이모지는 `aria-hidden="true"`로 처리하고 버튼과 탭의 접근 가능한 이름은 텍스트로 유지한다.
- 화면 확대 차단은 사용자가 요청한 기존 제품 정책으로 유지한다.

## 폰트 시스템

폰트 파일은 외부 CDN 없이 저장소에 WOFF2 형식으로 포함한다. 라이선스 전문도 폰트 자산과 함께 보관한다.

### 마루부리

- 용도: 주요 화면 제목, 카드의 감성 제목, 성경 구절, 영어동화 제목
- 기본 굵기: SemiBold
- 긴 본문, 입력값, 작은 보조문구와 숫자에는 사용하지 않는다.

### SUIT Variable

- 용도: 본문, 버튼, 탭, 입력창, 날짜, 시간, 수치, 차트 라벨
- 굵기: 400 본문, 600 보조 제목, 700 버튼과 강조
- 기존 Pretendard 선언과 시스템별 fallback 차이를 대체한다.

### 영문 소제목

- `FAMILY CARE GUIDE`, `BABY JOURNAL` 같은 eyebrow는 SUIT 700을 사용한다.
- 과도한 대문자 자간을 줄여 업무용 대시보드보다 동화책의 장 제목처럼 보이게 한다.

## 이모지와 손그림 역할

### 전역 탐색

- 일정 `🗓️`
- 성장 `🌱`
- 동화 `📖`
- 요청 `💡`
- 설정 `🎨`

기존 탭 구조와 텍스트는 유지하고 이모지의 크기, 기준선과 선택 상태만 통일한다.

### 주요 섹션

- AI 육아 도우미 `✨`
- 수유 `🍼`
- 수면 `🌙`
- 기저귀 `🧷`
- 성장 측정 `📏`
- 사진 `📷`
- 가족 공유 `👨‍👩‍👧`

섹션 제목 왼쪽의 둥근 색상 배지에 배치한다. 동일한 의미에는 앱 전체에서 같은 이모지를 사용한다.

### 행동과 상태

- 저장 `💾`
- 완료 `✅`
- 주의 `⚠️`
- 동기화 `☁️`
- 다시 시도 `↻`

버튼에는 이모지를 하나만 사용하고 버튼 텍스트를 대체하지 않는다. 성공·오류·주의 상태는 색상만으로 구분하지 않고 이모지와 문구를 함께 사용한다.

### 손그림 아이콘

큰 손그림은 페이지 대표 카드, 빈 화면, 첫 사용 안내에만 사용한다. 기존 가족 캐릭터와 충돌하지 않는 부드러운 수채화·종이 질감으로 통일하고, 작은 버튼 아이콘으로 축소하지 않는다.

## 화면별 적용

### 일정

- 오늘 카드와 캘린더 제목에 계절감 있는 작은 손그림을 한 개 사용한다.
- 일정 유형과 가족 구성원 정보에는 장식 이모지를 추가하지 않는다.

### 성장과 AI

- 성장일기, 기록하기, 돌봄 패턴, AI 육아 도우미, 최근 성장과 사진 섹션에 고정된 의미 이모지를 적용한다.
- AI 전략 본문과 출처에는 이모지를 넣지 않고 글자 크기와 대비를 우선 개선한다.
- 고정 성장 기록 버튼이 AI 콘텐츠를 가리는 문제는 별도 레이아웃 작업으로 해결한다.

### 동화

- 동화 장르 이모지는 유지하되 카드당 하나만 사용한다.
- 영문 제목과 한글 번역의 폰트 위계를 마루부리와 SUIT로 구분한다.

### 요청과 설정

- 요청 유형과 상태, 설정 카테고리 제목에만 의미 이모지를 사용한다.
- 설명, 글자 수와 저장 안내에는 이모지를 추가하지 않는다.

## 구현 순서

1. 마루부리와 SUIT WOFF2, 라이선스, `@font-face`를 추가하고 전역 폰트 역할을 연결한다.
2. 상단 탭과 주요 섹션 제목의 이모지를 의미 체계에 맞게 통일한다.
3. 별빛 밤의 제목·본문·영문 소제목 위계를 조정한다.
4. 390×844, 430×932, 768×1024, 1440×900에서 화면을 비교한다.
5. 다른 테마에서 글자 넘침, 버튼 높이, 카드 줄바꿈과 대비만 회귀 확인한다.
6. 이후 AI 가독성, 고정 버튼 겹침과 캘린더 작은 글자를 각각 독립 변경으로 처리한다.

## 범위 밖

- 모든 기존 아이콘을 한 번에 새 그림으로 교체하는 작업
- AI 답변 내용, Supabase 데이터, 인증, migration 또는 Edge Function 변경
- 새 CSS 보정 파일을 화면마다 추가하는 방식
- 확대 차단 정책 변경

## 성공 기준

- 제목은 동화책처럼 따뜻하고 본문·숫자·입력은 빠르게 읽힌다.
- 같은 기능은 모든 화면에서 같은 이모지를 사용한다.
- 이모지가 많아져도 정보 밀도와 접근 가능한 이름이 나빠지지 않는다.
- 폰트가 네트워크 연결 없이 동일하게 표시된다.
- 네 가지 기준 화면에서 겹침과 잘림이 없고 다른 테마도 기능적으로 유지된다.
