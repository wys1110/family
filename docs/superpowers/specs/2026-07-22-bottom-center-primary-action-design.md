# 하단 중앙 기본 액션 복원 설계

## 목표

일정·성장 화면의 공통 추가 버튼을 우측 상단의 인라인 배치에서 기존 하단 중앙 고정형 알약 버튼으로 복원한다. 사용자는 화면을 스크롤하는 동안에도 동일한 위치에서 새 일정이나 성장 기록을 시작할 수 있어야 한다.

## 승인된 동작

- 일정 탭에서는 하단 중앙에 `＋ 일정 추가`를 표시하고 기존 `openEventDialog()`를 호출한다.
- 성장 탭에서는 같은 위치에 `＋ 성장 기록`을 표시하고 기존 `openGrowthDialog()`를 호출한다.
- 일정과 성장 사이를 전환할 때 버튼의 문구, 접근 가능한 이름, 동작이 현재 탭과 일치해야 한다.
- 동화·요청·설정 탭의 기존 처리 방식은 변경하지 않는다.
- 성장 측정 내역 안의 `새 측정 기록` 버튼은 유지한다.

## 배치와 시각 디자인

- 버튼은 `body`의 직접 자식으로 이동해 모바일 Safari의 transformed-container 고정 위치 문제를 피한다.
- 화면 하단 중앙에 safe-area를 반영해 고정한다.
- 기존 `.fab`의 둥근 알약 모양, 의미형 색상, 최소 48px 터치 영역과 focus-visible을 유지한다.
- 우측 상단 또는 탭 바로 아래에 별도의 인라인 추가 버튼을 만들지 않는다.
- 긴 다가오는 일정과 성장 콘텐츠의 마지막 행이 버튼 뒤에 가려지지 않도록 본문에 고정 버튼 높이 이상의 하단 안전 여백을 둔다.
- AI 육아 도우미가 화면에 들어오면 기존 IntersectionObserver 안전 영역 정책으로 버튼을 숨긴다.

## 구현 범위

- `refresh-button.js`: `#addEventButton`을 다시 `body`의 직접 자식으로 이동한다.
- `refresh-button.css`: 하단 중앙 고정 배치와 safe-area, 반응형 간격, AI 안전 영역 선택자를 복원한다.
- 필요한 기존 레이아웃 CSS에 하단 안전 여백을 추가하되 화면별 임시 스타일 파일은 만들지 않는다.
- 모듈 캐시 버전을 갱신해 로그인된 브라우저에 변경이 즉시 전달되게 한다.

## 데이터와 기능 경계

- 일정·성장 다이얼로그, Supabase, 인증, 저장·수정·삭제 로직은 변경하지 않는다.
- 사용자 데이터나 현재 테마 설정을 생성·수정·삭제하지 않는다.
- `HANDOFF.md`와 `.superpowers/`는 수정하거나 커밋하지 않는다.

## 검증

- 자동 테스트로 버튼이 `body`에 배치되고 일정·성장 탭 문구와 기존 다이얼로그 연결이 유지되는지 확인한다.
- 모바일 너비에서 버튼이 중앙에 있고 마지막 다가오는 일정 행과 성장 콘텐츠를 가리지 않는지 로그인 브라우저에서 확인한다.
- 별빛 밤과 포근한 숲에서 색상·focus-visible·safe-area 배치를 확인한 뒤 별빛 밤으로 복원한다.
- `npm test`, `npm run check`, `git diff --check`를 실행한다.
